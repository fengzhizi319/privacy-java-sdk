# K-匿名产品需求文档

## 1. 背景

K-匿名通过准标识符泛化，确保发布数据中每条记录至少与 k-1 条其他记录不可区分，防止通过准标识符组合重新识别个体。

## 2. 目标

- 对单条记录按泛化层次进行准标识符泛化。
- 对整张表使用 Mondrian 算法进行多维分区泛化。
- 支持内置年龄/邮编/性别泛化层次。
- 支持自定义泛化层次。

## 3. 功能需求

| 编号 | 需求 | 优先级 |
|---|---|---|
| KAN-01 | 单记录泛化（anonymizeRecord） | P0 |
| KAN-02 | Mondrian 表格泛化（kAnonymizeTable） | P0 |
| KAN-03 | 内置泛化层次（age/zipcode/gender） | P0 |
| KAN-04 | 自定义泛化层次接口 | P1 |
| KAN-05 | 等价组大小 ≥ k 保证 | P0 |

## 4. 验收标准

- [ ] 单记录泛化后 QI 字段被泛化
- [ ] 非 QI 字段保持不变
- [ ] Mondrian 等价组大小 ≥ k
- [ ] 数据行数不足 k 时报错
- [ ] 自定义层次接口可扩展

## 5. 约束

- Mondrian 为贪心算法，不保证最优信息损失
- 内存中处理，超大数据集需分批
