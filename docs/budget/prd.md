# 隐私预算（Budget）产品需求文档

## 1. 背景

差分隐私的核心保证依赖于对隐私预算（ε, δ）的总量控制。预算模块为所有 DP 操作提供统一的预算记账与管控能力。

## 2. 目标

- 按 namespace 隔离预算，支持多业务场景。
- 提供 epsilon/delta 双维度预算管控。
- 支持内存与 SQLite 两种存储后端。
- 支持时间窗口自动重置。
- 线程安全的并发访问。

## 3. 功能需求

| 编号 | 需求 | 优先级 |
|---|---|---|
| BUD-01 | 按 namespace 创建/获取预算实例 | P0 |
| BUD-02 | 消耗预算（spend） | P0 |
| BUD-03 | 查询剩余预算（getRemaining） | P0 |
| BUD-04 | 预算耗尽时拒绝操作 | P0 |
| BUD-05 | 手动重置（reset） | P1 |
| BUD-06 | 时间窗口自动重置 | P1 |
| BUD-07 | SQLite 持久化 | P1 |

## 4. 验收标准

- [ ] 同 namespace 获取同一实例（单例）
- [ ] spend 后剩余预算正确减少
- [ ] 预算耗尽时抛出异常
- [ ] reset 后预算恢复
- [ ] 并发 spend 结果正确

## 5. 约束

- 内存模式下多进程不共享
- 采用简单顺序组合，未实现 RDP
