# 查询混淆运维手册

## 1. 概述

QoL 模块为纯本地计算，无外部依赖。运维重点在于 dummy 池管理与混淆效果评估。

## 2. 性能特征

| 操作 | 时间复杂度 | 说明 |
|---|---|---|
| 单条混淆 | O(D) | D=numDummies |
| 批量混淆 | O(N × D) | N=查询数 |

## 3. Dummy 池管理

- 池大小建议 ≥ numDummies × 3
- 自定义池内容应与业务领域相关
- 避免在池中包含真实敏感查询

## 4. 故障排查

| 问题 | 原因 | 解决方案 |
|---|---|---|
| dummy 大量重复 | 池太小 | 增大池或减少 numDummies |
| dummy 与查询无关 | 领域不当 | 使用自定义池 |

## 5. 安全注意事项

1. 日志中不要标记哪条是真实查询
2. 混淆后整批查询一起发送
3. numDummies 建议 ≥ 3
