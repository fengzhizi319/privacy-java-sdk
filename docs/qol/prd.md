# 查询混淆（QoL）产品需求文档

## 1. 背景

查询混淆通过向真实查询中混入虚假查询（dummy queries），使攻击者无法从查询日志中区分真实意图，保护用户查询行为隐私。

## 2. 目标

- 在真实查询中注入指定数量的 dummy 查询。
- 支持医疗与通用两个领域的内置 dummy 池。
- 支持自定义 dummy 池。
- 提供单条、批量、带详情三种调用方式。

## 3. 功能需求

| 编号 | 需求 | 优先级 |
|---|---|---|
| QOL-01 | 单条查询混淆 | P0 |
| QOL-02 | 批量查询混淆 | P0 |
| QOL-03 | 带详情混淆（QoLResult） | P1 |
| QOL-04 | 医疗/通用领域 dummy 池 | P0 |
| QOL-05 | 自定义 dummy 池 | P1 |
| QOL-06 | 语义槽位替换 | P1 |

## 4. 验收标准

- [ ] 混淆结果包含真实查询且位置随机
- [ ] dummy 数量等于 numDummies
- [ ] 医疗领域使用医疗池
- [ ] 自定义池优先于内置池
- [ ] 批量混淆各条独立

## 5. 约束

- dummy 池为静态内置列表
- 语义槽位替换为简单模板替换
